Annotation editor: real compiler help on the public type surface (F21). `AnnotationEditorConfig` is now generic — `AnnotationEditorConfig<TBody = W3CAnnotationBody, THostContext = unknown>` — and the extension hooks (`prepareDraft`, `beforeSave`, `onSelectionChange`) plus `AnnotationEditorRuntimeContext.selectedAnnotation` are typed as `W3CAnnotation<TBody>` instead of `any`. `W3CAnnotation` is now generic over its body and its `target`/`selector` are properly modelled: `target: W3CTarget | W3CTarget[]` with an open selector union (`FragmentSelector | PointSelector | SvgSelector | UnknownSelector`), plus index-signature escape hatches so host-specific fields survive a round-trip. The `AnnotationStorageAdapter` contract is fully typed (no `any`), and the internal skeleton markers (`__fullBodyLoaded`/`__bodyPreview`) are removed from the exported `W3CAnnotation` and expressed through a new `AdapterLoadResult` alias used only for `load()`/`hydrate()` returns. New exported types: `W3CTarget`, `W3CSelector`, `FragmentSelector`, `PointSelector`, `SvgSelector`, `UnknownSelector`, `AdapterLoadResult`. All generics carry defaults, so existing non-generic usage is unaffected; no runtime behavior changes.
